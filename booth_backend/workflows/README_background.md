# Background-removal workflows (AI, offline)

These turn the guest's spin clip into a **cutout composited on a new background** —
the marquee 2026 booth effect. No diffusion; just segment + recomposite, so they
run far faster than the style workflows.

Generated by `scripts/make_bg_workflows.py`. Same `VHS_LoadVideo` /
`VHS_VideoCombine` nodes as every other workflow, so the backend serves them by
name with no code change.

| workflow name | look |
|---|---|
| `bg_black` | guest on solid black ("spotlight") |
| `bg_white` | guest on solid white ("studio") |
| `bg_magenta` | guest on neon magenta |
| `bg_blur` | guest sharp, original background blurred |
| `bg_image` | guest on a custom still backdrop (`booth_background.png`) |

Use like any other: app sends the name, or
`flutter run --dart-define=BOOTH_WORKFLOW=bg_blur`, or `-F workflow=bg_blur` to
`POST /jobs`.

---

## ⚠️ UNVERIFIED — confirm on the Strix box before relying on these

ComfyUI cannot run on the Windows authoring machine (iGPU, no models), so these
JSONs were written from the **ComfyUI-RMBG v3.x docs**, not from a live export.
The graph *shape* is correct, but exact node `class_type`s and input keys can
drift between versions. **Validate once on the GPU box.**

### 1. Install the custom node
ComfyUI-Manager → Install Custom Nodes → **ComfyUI-RMBG** (by `1038lab`), restart.

### 2. Get the BEN2 model
Auto-downloads on first use to `ComfyUI/models/RMBG/`, or place manually in
`ComfyUI/models/RMBG/BEN2/`. BEN2 is chosen because it's **video-optimized**
(less frame-to-frame flicker than BiRefNet/RMBG-2.0). Note: RMBG-2.0 weights have
a non-commercial license; **BEN2 is fine** — don't switch the `model` to
`RMBG-2.0` for a paid event.

### 3. Reconcile node keys (the part that may need edits)
In ComfyUI, drop these nodes on a canvas, wire them as below, then **Save (API
Format)** and diff against the generated JSON. Fix any mismatched key names.

Expected graph (per file):
```
VHS_LoadVideo (1) ─► RMBG (2, model=BEN2, background=Alpha)
                        │ IMAGE(0)=cutout   MASK(1)=subject
   background source ───┤
     bg_black/white/magenta: EmptyImage (3)
     bg_blur:                ImageBlur (3) from the original frames
     bg_image:               LoadImage (3) ─► RepeatImageBatch (4)
                        ▼
                  ImageCompositeMasked (5)  destination=bg, source=cutout, mask
                        ▼
                  VHS_VideoCombine (6)
```
Nodes to verify specifically (most likely to differ by version/pack):
- **`RMBG`** — inputs `image, model, sensitivity, process_res, mask_blur,
  mask_offset, background, invert_output, refine_foreground`; outputs
  `[IMAGE, MASK]`. (If your pack exposes a separate **`BEN2`** node instead, swap
  `class_type` to that and keep `image` in / `[IMAGE, MASK]` out.)
- **`EmptyImage`**, **`ImageBlur`**, **`RepeatImageBatch`**,
  **`ImageCompositeMasked`** are ComfyUI core — usually stable, but confirm
  `ImageBlur` uses `blur_radius`/`sigma` and `ImageCompositeMasked` uses
  `destination/source/mask`.

### 4. For `bg_image`
Put your backdrop in ComfyUI's **input** folder as `booth_background.png` (or edit
the filename in `scripts/make_bg_workflows.py` and regenerate). Match its size to
the capture (512×512 by default) or the composite will offset.

### 5. Smoke test
```bash
python scripts/test_comfy.py            # if present, drives workflows on a sample
# or POST a sample clip:
curl -F file=@_samples/in.mp4 -F email=t@t.co -F consent=true -F workflow=bg_black \
     http://localhost:8000/jobs
```
Pass = the result shows the subject on the new background with clean edges. If
edges flicker, raise `mask_blur` (2→4) in the generator and regenerate.

## Regenerate / tune
Edit `PRESETS` in `scripts/make_bg_workflows.py`, then:
```bash
python scripts/make_bg_workflows.py          # rewrite all bg_*.json
python scripts/make_bg_workflows.py --list   # preview
```
