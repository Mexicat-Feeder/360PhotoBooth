# Motion / format workflows (the cornerstone 360 effects, all in ComfyUI)

The signature booth looks — slow motion, boomerang, vertical — as ComfyUI graphs
(no separate ffmpeg path). Light graphs: no checkpoint/diffusion, so they're fast.

Generated by `scripts/make_motion_workflows.py`. Same `VHS_LoadVideo` /
`VHS_VideoCombine` as every other workflow, so the backend serves them by name.

| workflow name | effect | needs RIFE node? |
|---|---|---|
| `slowmo` | RIFE frame-interpolated slow motion (the #1 effect) | **yes** |
| `boomerang` | forward→reverse loop (`VHS_VideoCombine` pingpong) | no |
| `vertical` | letterboxed 9:16 for Reels/TikTok | no |
| `slowmo_vert` | slow-mo + 9:16 combo | **yes** |

Use: app sends the name, or `-F workflow=slowmo` to `POST /jobs`, or
`flutter run --dart-define=BOOTH_WORKFLOW=boomerang`.

---

## Install (only slowmo* need this)
`slowmo` and `slowmo_vert` use **ComfyUI-Frame-Interpolation** (Fannovel16):
ComfyUI-Manager → Install Custom Nodes → **ComfyUI-Frame-Interpolation**, restart.
The `rife47.pth` model auto-downloads on first run.
`boomerang` and `vertical` use only core + VideoHelperSuite nodes (already
installed for the style workflows) — nothing extra.

## ⚠️ Verify node keys on the GPU box
Authored from docs, not a live export. Confirm on the Strix box (build once in the
UI → Save API Format → diff):
- **`RIFE VFI`** — inputs `frames, ckpt_name, multiplier, clear_cache_after_n_frames,
  fast_mode, ensemble, scale_factor`; output IMAGE. `multiplier=2` doubles frames
  (smoother slow-mo); raise to 4 for more. Note: `fast_mode` is ignored on rife
  ≥4.5. If VRAM is tight, lower `clear_cache_after_n_frames`.
- **`ImagePadForOutpaint`** (vertical) — core node; pads the 720² frames to 9:16
  (top/bottom borders). If your build names it differently, swap it for any
  pad/canvas node that outputs IMAGE. To fill the bars instead of black, replace
  with a blurred-copy composite (see `bg_blur` in README_background.md for the
  pattern).
- **`VHS_VideoCombine.pingpong`** (boomerang) — boolean; `true` appends the
  reversed clip. Standard VHS input.

## Tuning slow-mo
Two independent levers:
1. `RIFE VFI.multiplier` — how many in-between frames (2 = 2× smoother).
2. `VHS_VideoCombine.frame_rate` — lower than the source rate = slower playback.
   With `multiplier=2` and source 30 fps, set `frame_rate=30` for 2× slow-mo at
   full smoothness, or `frame_rate=15` for 4× (choppier).

## Regenerate / tune
Edit `PRESETS` in `scripts/make_motion_workflows.py`, then:
```bash
python scripts/make_motion_workflows.py          # rewrite all four
python scripts/make_motion_workflows.py --list
```
